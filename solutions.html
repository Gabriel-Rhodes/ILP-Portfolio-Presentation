It probably looks like nothing changed, but in the code, everything changed. We made i a variable (by using keyword let). Earlier the code had: for (i = 0;...) when we needed: for (let i = 0;...). This minor difference lets the computer register our i variable and means that our code will now work properly.

// JavaScript File let primes = []; function isPrime(num){... // Logic is if num inputed is not divisible by any prime in [primes], num is prime and returns true, else returns false. } //Assume all inputs will be positive or 0. function getPrimesToNum(num){  for (let i = 2; i <= num; i++){   if (isPrime(i)){    primes.push(i);   }  }  return primes } getPrimesToNum(24);

The issue is that we set i to 0 in the for loop earlier. How could this possibly break my code you ask? 0 and 1 aren't prime. By the logic in isPrime(), any input needs to not be divisible by every single element in primes. The first element will always be i. This is because i is not tested by any number. Therefore, we should let i equal the first prime number, or 2. If it equals 0, 0 and 1 will both count as prime when they are not and if we start at 3 or greater, we will ignore that 2 is prime. Therefore, in the for loop, i must start at 2.

*Side Note: you could and probably should use a while loop in this case, but both for and while work here. Also, when working with prime loops you generally start at 2.
